# Winning Strategy

The win condition is not “touch the most sponsor surfaces.”
The win condition is “look like a real system with disciplined claims.”

## Best Submission Stack

Lead with:

1. Synthesis Open Track
2. Protocol Labs: Agents With Receipts / ERC-8004
3. Protocol Labs: Let the Agent Cook
4. Virtuals: ERC-8183 Open Build
5. Celo: Best Agent on Celo
6. AgentCash / x402

## Why This Set Wins

### Open Track

- strongest system story
- broadest proof base
- easiest for both AI judges and humans to understand

### Protocol Labs ERC-8004

- trust is load-bearing, not decorative
- verifier and hook change execution behavior
- settlement writes can feed trust state back into the system

### Protocol Labs Let the Agent Cook

- the agent surface is already real
- package is published
- CLI supports `--json`
- canonical agent artifacts now exist

### Virtuals ERC-8183

- the product is already an agent commerce rail
- escrow, evaluator, and receipt logic are the core system

### Celo

- real testnet deployment breadth
- both success and failure flows are evidenced

### AgentCash / x402

- now backed by a real paid-request tx
- should be framed as truthful testnet proof, not the primary thesis

### Base

- now has a concrete public surface through the Base service directory
- should still be framed as a medium-strength secondary track, not as open-market proof

## Narrative To Use

Tell the story through scenarios:

### Scenario 1: Buyer agent needs an automation benchmark report

DealRail scans supply, ranks offers, and routes the job into escrow if the buyer wants stronger settlement guarantees.

### Scenario 2: Human operator needs a safer multi-party services desk

The browser desk shows live onchain jobs and blocks wrong-chain signing instead of silently misrouting transactions.

### Scenario 3: Agent operator needs a reproducible execution surface

The published package and JSON CLI mode let an agent do preflight, status, vend, and evidence lookup against the same live backend the browser uses.

### Scenario 4: Judge wants the Base-specific proof

The `/base` page and `GET /api/v1/base/agent-services` expose the Base-facing surfaces directly: provider directory, opportunity board, x402 proxy, Base job board, and settlement rail.

## What To Avoid

- claiming a fully live open marketplace
- describing the public Base directory as stronger than it is
- claiming sponsor-grade Uniswap or MetaMask proof
- inflating future Kairen stack plans into present-day proof
- describing curated demo supply as public market proof

## Highest-ROI Remaining Work

1. final demo video
2. tighter scenario-driven judge pitch
3. optional extra x402 proof if it is trivial
4. otherwise, resist new scope
